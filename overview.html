◈ User Story

As an ERP user, I want to create draft GL journal entries from Excel so that I can prepare transactions for manual review and posting by an accountant.

◈ Custom Function Description

WRITEBACKJOURNALDRAFT is an Excel custom function for creating draft GL journal batches in the ERP directly from a spreadsheet range. Each function defines a single line. Several of those functions placed together on a worksheet can define one GL journal batch. The function never posts the batch - it always lands in the On Hold status, and must be reviewed and posted manually inside the ERP system.

◇ Acceptance Criteria
Batch created on hold
The resulting batch in the ERP always has status On Hold; no release or post action is triggered.
Balance validation
Total debits must equal total credits across all lines; imbalanced input returns "Journal is not balanced: debits X.XX ≠ credits Y.YY" error.
At least one line required
An empty line range returns "At least one transaction line is required".
Return value
Returns "Draft batch #JE-00087 created" on success.
◻ Glossary
Batch
A batch is a group of journal lines (transactions) that have the same transaction date, ledger, and batch description.
Reporting Period
A reporting period is a predefined accounting period where the transaction date belongs (i.e. "Month Ending 31 March 2027").
